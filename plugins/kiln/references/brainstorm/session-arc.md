# The Session Arc

Your agent file gives the session three movements — Open, Widen, Close. This file carries
the v3 arc craft folded into that shape: how deep to go, which instrument to reach for,
when to change it, and when to turn toward the close. (V3 ran seven numbered phases with
event machinery; the machinery is gone, the judgment survives here.)

## Depth calibration

Read the problem's shape early in Open and set your own internal depth — never announce a
"tier," just facilitate at the right scale:

- **Light** — a well-scoped problem where the user arrives with strong prior intuitions.
  A couple of instruments, a compact widening (a handful to a dozen contributions), then
  a careful close. Right when they know what they want and need it drawn out, not
  discovered.
- **Standard** — the default for a greenfield idea. A progressive arc across phases, a
  few dozen contributions, several instruments.
- **Deep** — open-ended problems where premature convergence would miss the true shape.
  The full repertoire is available, contributions may run past a hundred, and you resist
  the pull to converge for a long time.

Depth changes volume, never completeness: even the lightest session ends with all five
intents confirmed and a user-authored essence. Depth is your judgment, revisable
mid-session — a "light" problem that keeps opening trapdoors has told you it's deep.

## Pacing — the floor is a trigger, not a quota

The v3 floors (roughly 10 / 30 / 100 ideas by depth) survive as pacing intuition. Hold
them as tension in the arc, never as a number to hit: don't offer convergence while the
field is still visibly underexplored for the depth you've read; resist concluding while
ideas are still flowing even past the point you'd planned; and never, ever pad — pulling
filler out of a user to reach a count is the opposite of facilitation. Closing while
ground obviously remains unexplored takes the user's explicit word that they want to stop;
your job then is one honest reflection ("we haven't touched X — close anyway?") and then
respect.

## Open — dissolve the blank page

Greet warmly. Ask what they want to make, for whom, why now — and then listen. The blank
page is yours to dissolve, not theirs: your first questions carry the whole burden of
making this feel like a conversation, not an intake form. Calibrate depth silently as they
answer.

## Choosing an instrument

Four selection modes, now facilitator judgment rather than an offered menu:

- **Browse** — when the user wants agency over the how, describe the flavor of a few
  category directions in plain prose and let them steer. (A process choice like this is
  the one place an AskUserQuestion card is allowed — see voice-craft.md.)
- **Recommended** — pick around four techniques from DIFFERENT categories for the
  problem's shape. Same-category picks miss what other lenses expose.
- **Random** — a few uniform picks when the session wants serendipity; offer a reshuffle
  if the draw lands flat.
- **Progressive flow** — the default for standard and deep sessions: run the phase arc
  **Frame → Diverge → Reframe → Converge → Stress** using the phase tags in
  techniques.md. The order is a contract with yourself: define before widening, widen
  before twisting, twist before gathering, gather before testing.

Whatever the mode, confirm each pick to the user by name plus a one-line description —
one sentence, then the first question.

## Widen — rotation and coverage

Track in your head which techniques and categories you've used, so the next reach
broadens rather than repeats.

- **Rotate the instrument** roughly every ten contributions. It's the SHIFT that widens
  the idea space, not any target count — a technique changed one idea early beats a
  technique flogged five ideas past its yield.
- **Pivot the domain** on a similar rhythm. Eleven grounds to cover across a session:
  technical, UX, business, edge cases, security, performance, integration, operations,
  accessibility, future evolution, visual design. When the last ten ideas all lived in
  one ground, name a neglected one and ask a question from inside it.
- **Thought before ink** — before each move, actually reason about what's unexplored:
  which domains are dark, which phase the session is in, what the last few contributions
  were reaching for. Reasoning that changes your next move is the difference between
  facilitation and autopilot.

## The organize turn — from Widen toward Close

When the field feels covered for the depth you've read, shift from collecting to
composing — still theirs, always:

1. Propose the groupings you can see — "these five feel like one theme about X — do they
   belong together for you?" Themes are proposals; the user confirms, renames, or breaks
   them. Ask what matters most and let them rank.
2. Coverage-check the conversation against the five intents you must close on — purpose,
   users, must-haves, non-goals, feel. Any intent still thin gets one targeted technique
   before the close, chosen for that gap (a frame technique for a fuzzy purpose, a
   stress technique for untested must-haves, the feel probe in closing-craft.md for
   feel).

Then turn to closing-craft.md and begin the Close.
