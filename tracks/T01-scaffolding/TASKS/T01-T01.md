# T01-T01: Create package.json

## GOAL
Create the NPM package manifest for kiln-dev with correct name, version, bin entry, files whitelist, and zero runtime dependencies.

## ACCEPTANCE_CRITERIA
- AC-01 (DET): `npm pack --dry-run` succeeds and lists agents/, skills/, hooks/, bin/, templates/ in output

## FILES
- path: package.json
  action: add
  rationale: NPM package manifest — defines the package identity, bin entry, and files whitelist

## COMMANDS
- `node -e "const p = require('./package.json'); console.log(p.name, p.version, p.bin);"` — validate JSON and key fields
- `npm pack --dry-run` — verify package can be packed

## SUMMARY
Create package.json at the repo root with name "kiln-dev", version "0.1.0". Set the bin field to `{"kiln-dev": "bin/install.js"}`. Set the files array to `["agents/", "skills/", "hooks/", "bin/", "templates/"]` to whitelist only product directories for npm publish. Set license to "MIT". Do NOT add any dependencies or devDependencies. Add a description field: "Multi-model orchestration workflow for Claude Code". Set the main field to "bin/install.js". Ensure the JSON is valid and well-formatted.

## ESTIMATED_DIFF
~20 lines

## RISKS
- Incorrect files whitelist could exclude product directories from the published package
- Missing bin entry would break `npx kiln-dev`
