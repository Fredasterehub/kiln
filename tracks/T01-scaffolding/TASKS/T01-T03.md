# T01-T03: Create stub installer

## GOAL
Create a minimal bin/install.js that parses --help and --repo-root arguments, prints usage information, and exits cleanly. No real install logic yet — that is Track T09.

## ACCEPTANCE_CRITERIA
- AC-03 (DET): `node bin/install.js --help` exits with code 0 and prints usage to stdout

## FILES
- path: bin/install.js
  action: add
  rationale: Entry point for `npx kiln-dev` — stub with argument parsing skeleton

## COMMANDS
- `node bin/install.js --help` — must exit 0 and print usage

## SUMMARY
Create bin/install.js with a Node.js shebang line (`#!/usr/bin/env node`). Parse process.argv for `--help` and `--repo-root <path>` arguments using only Node.js built-ins (no external argument parsing libraries). When `--help` is passed, print usage information showing: the package name (kiln-dev), a brief description, available options (--help, --repo-root), and example usage (`npx kiln-dev` or `npx kiln-dev --repo-root /path/to/project`). Exit with code 0 after printing help. When run without --help, print a message like "kiln-dev installer — not yet implemented" and exit 0. Make the file executable (chmod +x is set via git, but include the shebang). Use zero external dependencies — only Node.js built-ins (process, path).
